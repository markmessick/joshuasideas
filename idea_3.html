Idea #3: Perform a Monthly Analysis on the Cause of Reservices

What if a report automatically showed up in your inbox each month with an in-depth analysis on every reservice from that month?

The current reservice report is super good and provides really useful data on the field experts and target pests that contribute towards reservices. This report would look at other pieces of data that can only be obtained through the Pestroutes API.

It would try to uncover patterns that haven't been observed yet. For example, does the time of day a Quarterly service is performed contribute to it's likelihood of becoming a reserivce? How about the length of the Quarterly service? Or the application rate of each product?

Once the report is set up the first time, it could automatically generate each month with updated trends and patterns.
