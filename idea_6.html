Idea #6: Automatically Generate the Monthly Reservice Report

What if Office Experts never had to slave away over the Reservice Report ever again?

This one is definitely ambitious, but I've got super high hopes.

Pestroutes has provided a very handy API (check it out here) so that developers can use a programming language (such as Python) to remotely scrape information from their system. The API can be used to pull all kinds of bulk information on customers, appointments, and a bunch of other stuff.

The hope is that I could create a script in Python that would reach out to the API and automatically pull all of the same information we're manually entering into the Reservice report right now.

However, sometimes APIs are tricky and impose certain rules and restrictions. If that's the case and it prevents us from accessing the information we need, we do have one other option. Lately I've been learning how to use a Python module named BeautifulSoup, which is a web scraping program that vists a website, and then actually digs down into the code and pulls whatever information I tell it to. This one is quite a bit more involved and would take a lot more time, but is still definitely possible.
